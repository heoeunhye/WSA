# 메타데이타 관련 요소

메타데이타 관련 요소는 정보를 정의하는 요소로 head 요소 내에서 자식 요소로 마크업 할 수 있습니다.

# meta 요소

meta 요소는 head 요소 안에 위치하며, 문서의 다양한 문서 정보를 나타낼 때 사용합니다. meta 요소로 지정한 정보들은 화면에 표시되지 않지만 기기가 해당 정보를 이해하고 분석할 수 있도록 정보(데이터)를 제공합니다. 보통 name 속성과 content 속성을 함께 사용하며, name 속성에 값을 지정하고 지정된 값에 해당하는 정보를 content 속성에 명시합니다.

📌문자 인코딩 지정하기

meta 요소에 charset 속성을 사용하여 웹브라우저에서 한글이나 영어와같은 문자를 처리하기 위해 문자 코드 세트(인코딩)를 지정할 수 있습니다. 이때 charset 속성의 경우 HTML5부터 추가되었으며 기존에는 content 속성에 지정하였습니다. 현재 새롭게 간소화된 방식과 기존에 제공되던 방식 모두 허용되고 있습니다. 문자 인코딩 지정의 경우 meta 요소 중 가장 먼저 선언을 권장 합니다.





 

 








<!DOCTYPE html>
<html lang="ko-KR">
<head>
 <!-- HTML5에 추가된 방식 -->
 <meta charset="utf-8">
 <!-- 기존 방식 -->
 <meta http-equiv="content-type" content="text/html; charset=euc-kr">
 <title>문자 인코딩 지정하기</title>
</head>
<body>
 <h1>문자 인코딩</h1>
 <p>문자 인코딩 지정은 HTML5 방식과 기존 방식 모두 허용됩니다.</p>
</body>
</html>

📌키워드 지정하기

meta 요소를 이용하면 웹 페이지 홍보의 수단으로 검색 키워드를 지정할 수 있습니다. 웹 페이지에 검색 키워드를 지정하면 검색할 때 웹 사이트가 상위에 노출되는 효과를 얻을 수 있습니다. meta 요소를 이용하여 검색 키워드를 지정할 때는 검색 키워드를 콤마(,)로 구분하여 선언합니다.






 









<!DOCTYPE html>
<html lang="ko-KR">
<head>
 <meta charset="utf-8">
 <title>검색 키워드 지정</title>
 <meta name="keywords" content="웹접근성,웹표준,HTML,웹콘텐츠 접근성 지침">
</head>
<body>
 <h1>검색 키워드</h1>
 <p>meta 요소를 이용하면 웹 페이지 홍보의 수단으로 검색 키워드를 지정할 수 있습니다. 웹 페이지에 검색 키
  워드를 지정하면 검색할 때 웹 사이트가 상위에 노출되는 효과를 얻을 수 있습니다. meta 요소를 이용하여 검
  색 키워드를 지정할 때는 검색 키워드를 콤마(,)로 구분하여 선언합니다.</p>
</body>
</html>

📌다양한 문서 정보 지정하기

화면에 보이는 콘텐츠는 아니지만 웹 페이지의 다양한 문서 정보를 지정할 때도 meta 요소를 사용할 수 있습니다. 문자 코드 세트와 키워드 외에도 subject를 이용하여 문서 제목 정보를 제공하고, description값으로 웹 페이지의 요약 정보, 제작자 정보, 저작권 정보 및 검색 로봇 제어 여부 등의 문서 정보를 지정할 수 있습니다.







 
 
 
 
 
 







<!DOCTYPE html>
<html lang="ko-KR">
<head>
 <meta charset="utf-8">
 <title>문서 정보 삽입하기</title>
 <meta name="keywords" content="웹 접근성, 웹 표준, 품질마크, 웹 콘텐츠 접근성 지침">
 <meta name="subject" content="웹 접근성 전문가 소개">
 <meta name="description" content="웹 접근성에 대한 개요와 웹 접근성 품질마크 소개 및 
 한국형 인터넷 웹 콘텐츠 지침에 대한 내용입니다.">
 <meta name="author" content="seulbinim@gmail.com(Teresa Kim), 김데레사">
 <meta name="robots" content="index,follow">
 <meta name="copyright" content="copyrights 2010 WAP corp.">
 </head>
<body>
 <h1>다양한 문서 정보 지정</h1>
 <p>화면에 보이는 콘텐츠는 아니지만 웹 페이지의 다양한 문서 정보를 지정할 때도 meta 요소를 사용할 수 있습니다. 문자 코드 세트와 키워드 외에도 subject를 이용하여 문서 제목 정보를 제공하고, description값으로 웹 페이지의 요약 정보, 제작자 정보, 저작권 정보 및 검색 로봇 제어 여부 등의 문서 정보를 지정할 수 있습니다.</p>

📌반응형 설정을 위한 viewport 설정하기

웹브라우저들은 뷰포트(viewport) 영역에 페이지를 렌더링합니다. 이때 뷰포트는 화면 상에 보이는 영역을 의미하는데 처음 모바일 사파리(Safari)의 경우 meta 요소에서 viewport를 지정하여 화면 크기를 조정할 수 있게 제공 하였습니다. 현재 대다수의 웹브라우저 들이 이를 지원하고 있습니다.






 









<!DOCTYPE html>
<html lang="ko-KR">
<head>
 <meta charset="utf-8">
 <title>뷰포트 지정하기</title>
 <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>
<body>
 <h1>뷰포트 지정하기</h1>
 <p>뷰포트는 화면 상에 보이는 영역을 의미하는데 처음 모바일 사파리(Safari)의 경우 
  meta 요소에서 viewport를 지정하여 화면 크기를 조정할 수 있게 제공 하였습니다. 
  현재 대다수의 웹브라우저 들이 이를 지원하고 있습니다.</p>
</body>
</html>

🌑meta 요소의 viewport 관련 선언으로 지정할 수 있는 값은 다음과 같습니다.

width
viewport의 가로 크기를 지정합니다. 이때 px, em, %등 다양한 단위의 값을 지정할 수 있으며 모바일 디바이스의 물리적인 크기로 지정할 경우 device-width와 같은 키워드를 사용할 수도 있습니다.
height
viewport의 세로 크기를 지정합니다.
initial-scale
초기 렌더링 비율을 의미하며 1을 지정할 경우 100%를 의미합니다.
user-scalable
사용자가 확대/축소 기능을 사용할 수 있도록 설정할 수 있으며 yes, no 값을 사용할 수 있습니다. 이때 기본값은 yes 입이다.
maximum-scale
user-scalable 값이 yes일 때 화면을 확대할 수 있는 최대 비율을 의미합니다.
minimum-scale
user-scalable 값이 yes일 때 화면을 축소할 수 있는 최대 비율을 의미합니다.

# title 요소

웹 문서의 제목을 선언할 때 사용하는 요소로, 문서마다 유일한 내용으로 구성해야 합니다. 시각 장애인의 경우에는 화면 낭독기를 이용하여 콘텐츠를 탐색합니다. 따라서 title 요소는 문서를 구별할 수 있는 첫 번째 관문이라고 할 수 있습니다. 또 title 요소는 즐겨찾기 및 북마크에 해당 웹 문서를 추가할 경우에 사용되는 북마크 이름으로도 활용되므로, 같은 title을 가진 웹 문서가 여러 개 있어서는 안됩니다.





 









<!DOCTYPE html>
<html lang="ko-KR">
<head>
 <meta charset="utf-8">
 <title>웹 문서의 제목</title>
</head>
<body>
 <h1>웹 문서의 제목</h1>
 <p>웹 문서의 제목을 선언하려고 할 때 사용하는 요소로, 문서마다 유일한 내용으로 구성해야 합니다. 
  시각 장애인의 경우에는 화면 낭독기를 이용하여 콘텐츠를 탐색합니다. 
  따라서 title 요소는 문서를 구별할 수 있는 첫 번째 관문이라고 할 수 있습니다.</p>
</body>
</html>

👍페이지와 웹 사이트 정보 함께 표기하기

title 요소는 문서와 문서 사이를 이동할 때 해당 웹 페이지가 어떤 정보를 담고 있는지를 제일 먼저 알려 줍니다. 따라서 현재 페이지가 어떤 내용을 담고 있는지를 문서 제목을 통해 쉽게 파악할 수 있어야 하고, 다음 페이지가 어떤 내용을 담고 있는지도 함께 표시하는 것이 좋습니다. 최근에는 대부분의 웹 브라우저가 탭 브라우징을 제공하고 있습니다. 예를 들어‘공지사항’페이지의 경우 단순하게‘공지사항’이라고만 표기하면, 한 웹 브라우저에 여러 웹 사이트를 탭으로 띄었을 경우 ‘공지사항’ 콘텐츠가 담긴 페이지인 것은 알지만 어느 웹 사이트의 공지사항 인지는 알 수 없게 됩니다.

<title>묻고 답하기 | WebCafe</title> // WebCafe라는 웹 사이트의 묻고 답하기 페이지
1

위의 예처럼 탭 브라우징을 생각하면‘<title>WebCafe | 묻고 답하기</title>’와 같이 웹 사이트명이 먼저 나오는 것이 좋다고 생각할 수 있습니다. 하지만 시각이 아닌 음성에 의해 정보를 순차적으로 제공하는 환경이라면 현재 접속한 곳이 어떤 웹 사이트인지를 인식한 상태에서 링크를 클릭했을 때 원하는 정보에 맞게 접근했는지를 문서 제목을 통해 가장 먼저 확인할 수 있습니다. 문서 제목이 없을 경우에는 화면에 표시되는 내용을 보고 현재 문서의 정보를 알 수 있지만, 시각장애인의 경우에는 일일이 페이지의 내용을 탐색하여 파악해야 하므로 불편합니다.

👎의미없는 장식문자 사용하기

제목을 좀 더 멋지게 보이려는 목적으로 title 요소 안에 ‘:::’ 등의 장식문자를 사용한 경우입니다.

<title><title>:::::::::::::::: ▒ 우리회사에 방문하신 것을 환영합니다. :::::::::::::::: ▒</title>
1

이와 같이 장식문자를 과도하게 사용하면 처음에는 멋지게 보일 수 있지만, 스크린 리더와 같은 보조기기를 통해 접근할 때 ‘콜론 콜론 콜론 콜론(반복) 우리 회사에 방문하신 것을 환영합니다 콜론 콜론 콜론…’으로 읽힐 수 있고, 탭 에서는 정보 텍스트가 콜론 기호에 밀려 안 보이게 됩니다.

제목은 해당 페이지의 내용을 이해할 수 있게 간결하게 명시하거나 웹 사이트의 첫 화면(메인)일 경우 “우리 회사에 오신 것을 환영합니다”와 같이 인사 문구 정도를 함께 쓸 수 있습니다. 웹 문서의 검색 최적화를 위해 title은 간결하고, 현재 페이지의 정보를 잘 나타낼 수 있도록 지정해 주시기 바랍니다.

여기서 잠깐

title 요소에 사용하는 장식 문자

과거에 어떤 에이전시의 경우에는 웹 사이트의 title 장식 문자가 정해져 있어서 그것을 자사에서 제작한 웹 사이트라는 고유의 표시로 사용하기도 했습니다. 하지만 웹 접근성을 유지하기 위해 이 경우에는 앞에서 배운 meta 요소를 이용하여 제작사 정보 등으로 변경하여 넣는 것이 좋습니다.

# link 요소

link 요소는 외부 문서를 연결하거나 다른 문서와의 관계를 지정할 수 있습니다. link 요소를 사용하여 스타일 시트를 HTML 문서에 연결하거나 파비콘(favicon) 등을 지정하는 경우가 link 요소의 대표적인 사용 예라고 할 수 있습니다.

📌파비콘(favicon) 설정하기

파비콘(favicon)은 웹브라우저가 웹페이지의 제목과 함께 보여주는 이미지로 ico 확장자를 가집니다. 파비콘은 일반적으로 16x16 픽셀을 사용하고 있으나 모바일 디바이스가 등장함에 따라 좀 더 큰 이미지의 파비콘을 사용하게 되었습니다. 또한 ico 형식의 단점으로 투명도를 지원하지 않는 문제가 있는데 최근에는 ico 이외에 png, gif, jpeg 및 svg와 같은 다양한 형식을 사용할 수 있습니다. 다만 IE10 이하의 경우 ico 형식 만을 제공하기 때문에 하위 호환성을 고려할 경우 주의가 필요합니다.






 
 
 







<!DOCTYPE html>
<html lang="ko-KR">
<head>
 <meta charset="utf-8">
 <title>파비콘 설정하기</title>
 <link rel="icon" href="favicon16.png" sizes="16x16" type="image/png">
 <link rel="icon" href="favicon48.png" sizes="48x48" type="image/png">
 <link rel="icon" href="favicon192.png" sizes="192x192" type="image/png">
</head>
<body>
 <h1>파비콘이란?</h1>
 <p>파비콘(favicon)은 웹브라우저가 웹페이지의 제목과 함께 보여주는 이미지 입니다.</p>
</body>
</html>

📌스타일시트 연결하기

link 요소로 외부 리소스 중 CSS 파일을 지정할 수 있습니다. CSS는 스타일 시트로 HTML 요소에 디자인을 적용할 때 사용합니다. 다음은 link 요소를 사용하여 외부에 별도로 생성한 CSS 파일을 지정한 사례 입니다.






 








<!DOCTYPE html>
<html lang="ko-KR">

# style 요소

요소 등에 대한 스타일 정보를 지정할 수 있습니다. 이때 style 요소 내에서는 CSS 문법을 사용하여야 합니다.

style 요소는 특정 웹 페이지에만 스타일을 적용하고자 할 때 사용할 수 있으며 HTML5에서는 scoped 속성이 새롭게 추가되었습니다. scoped 속성은 특정 문서 내 모든 콘텐츠가 아닌 특정 콘텐츠 블록에만 CSS를 적용할 수 있도록 합니다. 그러나 아직 scoped 속성을 지원하는 웹브라우저가 없습니다.






 
 
 
 
 
 
 
 
 






 
 
 
 
 
 







<!DOCTYPE html>
<html lang="ko-KR">
<head>
 <meta charset="utf-8">
 <title>style 요소</title>
 <style type="text/css">
  body{
   color:#000;
   background-color:#fff;
   font-size:12px;
   line-height:1.2;
   font-family:'돋움','dotum',sans-serif;
  }
</style>
</head>
<body>
 <h1>문서 전체에 적용</h1>
 <p>배경 색상은 흰색, 글자 색상은 검정색, 글자크기는 12px, 줄높이는 1.2 
  글꼴은 돋움체로 스타일이 적용됩니다.</p>
 <div>
  <style scoped>
   body{
   color:#fff;
   background-color:#000;
  }
  </style>
  <h1>특정 영역에 적용</h1>
  <p>배경 색상은 검정색, 글자 색상은 흔색으로 적용됩니다.
   그러나 아직 지원하는 웹브라우저가 없습니다.</p>
 </div>
</body>
</html>
