# 텍스트 레벨 요소

# a 요소

a 요소는 하이퍼링크를 지정할 때 사용하는 요소로, HTML5에서는 변경된 개념과 함께 추가된 속성이 존재합니다. 우선 HTML4.01와 XHTML1.0에 비해 달라진 개념으로는, 기존에는 블록 전체를 a 요소에 포함하여 하이퍼링크를 지정할 수 없었지만, HTML5에서는 블록 전체를 포함하여 하이퍼링크를 지정할 수 있습니다.

또한 href 속성이 없는 a 요소의 사용도 가능합니다. 이러한 형태를 플레이스 홀더(Placeholder)라고 하며, a 요소를 플레이스 홀더로 사용하고자 할 경우, 처음 HTML 문서 렌더링 시에는 링크가 생성되지 않았다가 자바스크립트를 사용하여 동적으로 href 속성을 지정할 경우에 사용합니다.

새롭게 추가된 속성으로는 download 속성이 있으며, 참조하는 리소스가 하이퍼링크로 이동하는 것이 아니라 리소스를 다운로드해야 하는 경우에 사용할 수 있습니다. 그러나 아직 download 속성을 지원하는 웹 브라우저는 없습니다.

👎구체적이지 않은 링크 콘텐츠

사이트를 이용하다 보면‘자세한 사항은 이곳을 클릭하여 주세요.’라는 문구를 종종 봅니다. 이 경우 ‘이곳’이라는 단어에 링크가 연결되어 있어서 클릭하면 연결된 페이지로 넘어갑니다. 그렇다면‘이곳’은 어디일까요? 사용자는 막연하게 상세 정보가 있을 것이라 생각하고 링크를 클릭하지만,‘이곳’이라는 텍스트만으로는 링크의 의미를 제대로 전달하지 못합니다.

이렇게 링크가 연결되는 경우는 의미에 맞게 적용해야 합니다. ‘자세한 사항은 고객 문의 게시판에서 확인해 주세요.’와 같이 링크 자체만으로도 의미를 전달할 수 있어야 합니다.

📌절대 경로와 상대 경로 알아보기

웹 문서에서 링크를 설정하거나 이미지를 삽입할 때 또는 외부 스타일시트 문서를 연결하는 작업 등을 할 때는 파일 경로를 지정해야 합니다. 파일 경로 지정 방식에는‘절대 경로’와‘상대 경로’가 있습니다.

절대 경로는 전체 구조 내에서의 위치를 표시하는 방법입니다. 만약 로컬 기준으로 절대 경로를 지정하는 경우, file:///프로토콜을 시작으로 연결하려는 파일의 전체 경로를 명시합니다.

위의 예시에서 logo_top.gif 이미지를 index.html 문서에 삽입할 때 다음과 같이 절대 경로를 지정할 수 있습니다.

- 형식
<img src="file:///루트 폴더(C:\)/폴더 경로/파일명" alt="대체 텍스트">
- 예시
<img src="file:///C:\/images/common/logo_top.gif" alt="로고">

상대 경로는 작업하고 있는 파일의 현재 위치를 기준으로 경로를 지정하는 방법입니다. 상대 경로의 형식으로 정리해 보면 다음과 같습니다.

  1. 연결하려는 파일이 현재 편집 중인 문서와 같은 폴더에 위치한 경우
    (현재 편집 중인 문서는 index.html이고 연결하려는 파일은 같은 폴더에 위치한 information.html)
- 형식
<a href="파일명">연결할 문서</a> 또는 <a href="./파일명">연결할 문서</a>
- 예시("./"는 같은 폴더를 의미)
<a href="information.html">상세안내</a> 또는 
<a href="./information.html">상세안내</a>

    (현재 편집 중인 문서는 index.html이고 연결하려는 파일은 상위 폴더에 위치한 information.html)

그러나 링크의 title 속성은 스크린 리더에서 기본으로 읽어 주지 않기 때문에 설정을 바꾸지 않는 한, 사용자는 이 부분을 알지 못하고 지나칠 수 있습니다. 또 이미 텍스트로 제공하고 있는 같은 정보를 title로 중복 제공하는 문제점이 발생합니다. title과 콘텐츠의 중복을 막고 디자인을 변경하지 않으려면 새롭게 등장한 WAI-ARIA의 aria-labelledby 속성을 사용할 수 있습니다.

WAI-ARIA의 aria-labelledby 속성을 사용하기 위해서 가장 먼저 다운로드 링크의 목적을 알 수 있는 정보 콘텐츠를 span 요소로 그룹화 한 후 고유의 id 값을 지정합니다. 그 후 하이퍼 링크 요소인 a 요소에 aria-labelledby 속성을 추가하고 연결하고자 하는 id 값을 지정하면 됩니다. 이렇게 연결할 경우 스크린 리더에서는 "다운로드 링크"가 아닌 "2009년 7월 목록 다운로드 링크"라고 읽어주기 때문에 링크의 목적지를 좀 더 명확히 이해할 수 있게 됩니다.

# em 요소

em 요소는 강조하고자 하는 콘텐츠를 마크업할 때 사용합니다. 의미상 강조하여 사용자에게 어필하고 싶은 콘텐츠에 사용하는 것이 적합 합니다.

# strong 요소

strong 요소는 HTML4.01이나 XHTML1.0에서 강조의 의미를 가지며, em 요소와의 뚜렷한 차이점이 없다는 점으로 인해 구분이 모호한 요소였지만, HTML5에서는 그 의미가 변경되었습니다. HTML5에서 strong 요소는 제작자의 의도에 따라 중요한 콘텐츠임을 나타내고자 하는 경우에 사용할 수 있습니다. 또한 strong 요소를 중첩하여 사용할 경우, 중요도의 강도를 더 높여 표현할 수 있습니다.

Codepen으로 보기

# i 요소

i 요소는 HTML4.01이나 XHTML1.0에서 기울임꼴 형태의 글자로 장식하기 위한 용도였으며, 비 추천 요소로 사용이 권장되지 않았지만 HTML5에서는 분위기 전환이나 관례상 흔히 이탤릭체를 사용하는 격언, 기술 용어, 분류 용어 등에 사용할 수 있도록 의미가 변경되었습니다.

Codepen으로 보기

# b 요소

b 요소는 HTML4.01이나 XHTML1.0에서 굵은 글자로 장식하기 위한 용도로, 비추천 요소였지만 HTML5에서는 다른 콘텐츠와 구별하기 위한 의도를 가지는 경우에 사용할 수 있도록 의미가 변경 되었습니다. b 요소의 기본 스타일은 굵은 글자의 형태를 가지고 있지만 CSS를 활용하여 다양한 디자인으로 표현할 수 있습니다. 단 b 요소는 strong 요소나 em 요소처럼 중요도나 강조의 의미를 가지고 있지는 않습니다.

Codepen으로 보기

# q 요소

문장 형태의 짧은 인용문의 경우 q 요소를 이용할 수 있습니다. q 요소는 인라인 요소와 텍스트를 포함할 수 있습니다. 인용문의 경우 출처를 명시하려고 할 때 cite 속성으로 인용한 곳을 지정할 수 있으며 특히 q 요소를 사용할 경우 웹표준을 지원하는 웹 브라우저에서 인용의 표현으로 따옴표가 함께 렌더링 됩니다(인터넷 익스폴로러 7 이하는 지원하지 않음.)

Codepen으로 보기

# abbr 요소

abbr 요소는 축약어(abbreviation)를 마크업할 때 사용하는 요소 입니다. abbr 요소는 WWW(World Wide Web)처럼 스펠링을 하나씩 발음하는 경우나 Apartmention을 Apt.라고 줄여서 단어의 자음이나 모음만 맞추는 등 긴 단어를 단순하게 줄여 사용하는 것을 말합니다.

이렇게 축약된 약어의 정식 명칭을 부연할 경우, title 속성을 사용할 수 있으며 title 속성에 지정한 내용은 풍선 도움말 형식으로 웹 브라우저 화면에 나타납니다.

Codepen으로 보기

# ins, del 요소

콘텐츠를 추가하거나 삭제하고자 할 때는 ins 요소와 del 요소를 사용할 수 있습니다. ins 요소와 del 요소의 경우 블록 콘텐츠 단위로 추가하거나 삭제할 때, 인라인 콘텐츠 단위로 추가하거나 삭제할 때 모두 사용할 수 있다는 특징이 있습니다.

# small 요소

small 요소는 small이라는 단어에서 주는 의미처럼 작은 글씨로 표현하고자 하는 주석 등의 콘텐츠를 마크업할 때 사용합니다. 이때 small 요소로 마크업할 수 있는 콘텐츠의 범위는 섹션 전체나 목록 등의 블록 전체가 아닌 인라인 수준의 짧은 콘텐츠에 적합합니다.

# time 요소

time 요소는 HTML5에 새롭게 추가된 요소로, 컴퓨터가 이해할 수 있는 날짜 및 시간 정보를 마크업 할 수 있습니다. 이때 시간은 24시간 방식, 날짜는 GMT 표준시 형식으로 표현해야 합니다. time 요소에 규정된 형식은 "2013-10-04T11:30:00+09:00"으로, 날짜는 연월일을 하이픈(-)으로, 시간은 콜론(:)으로 표현하며, 날짜와 시간 정보를 구분할 때는 시간 정보 앞에 T를 붙입니다.

time 요소를 사용할 때 날짜나 시간을 인간이 이해하기 쉬운 형태로 마크업할 수도 있습니다. 예를 들어 2019년 10월 4일이라는 표현이 2019-10-04 형식의 표현보다 더 이해하기 쉬운 날짜 형식이기 때문입니다. 만약 이러한 형태로 마크업할 경우 시스템 날짜 형식은 datetime 속성으로 제공하면 됩니다.
