# CSS 디자인

# CSS란?

CSS 또는 캐스케이딩 스타일 시트(Cascading Style Sheet)는 마크업 언어가 실제로 표시되는 방법을 기술하는 언어로, HTML과 XHTML에 주로 쓰이며, XML에서도 사용할 수 있습니다. 또한 CSS는 W3C의 표준이며, 레이아웃과 스타일을 정의할 때의 자유도가 높은 언어입니다.

스타일은 웹의 독창적인 개념이 아니라 웹 이전부터 워드프로세스 등에 사용했던 개념으로 웹표준에 대한 관심이 높아지면서 CSS를 이용한 디자인도 함께 주목받고 있습니다. 왜냐하면 CSS가 기존 디자인 방식의 체질을 개선시킬 수 있기 때문입니다. 특히 CSS는 테이블 방식의 레이아웃 때문에 마크업 코드가 복잡해지는 문제를 해결할 수 있고, 표현과 관련된 요소(font, b, i 요소 등)의 사용을 제한하면서 풍부한 CSS 속성으로 대체할 수 있는 장점이 있습니다.

또 CSS 방식의 디자인은 웹 사이트의 성능 향상에 기여하며 웹 접근성이나 웹 사용성을 높이는 데도 효과적입니다. 간혹 인터넷 익스플로러 6처럼 오래된 웹 브라우저에서 CSS를 제대로 지원하지 않기 때문에 문제가 발생하기도 하지만, 이런 경우는 다양한 CSS 핵(hack)으로 보완할 수 있습니다. 그러나 구식 웹 브라우저를 배려하고자 일부 CSS만 이용하는 것은 집을 지을 수 있는 도끼를 가지고 과일을 깎는 것처럼 어리석은 일입니다.

인터넷 익스플로러의 경우 버전 7부터 부분적으로 웹 표준을 지원하기 시작했고, 버전 8에서는 전면적으로 웹 표준을 지원하고 있습니다. 이 때문에 인터넷 익스플로러는 6, 7, 8 버전이 각각 다르게 렌더링하는 문제가 발생하여 크로스 브라우징이 까다롭게 되었습니다. 그러나 미래를 위한 마이크로소프트사의 선택은 옳았다고 봅니다. 과도기적인 불편함을 넘어서는 순간 CSS의 디자인은 강력한 효과를 발휘하게 되므로 이제 더 이상 CSS를 이용한 디자인을 외면할 수는 없습니다. 또 다양한 웹 브라우저 환경에서 CSS는 더욱 중요해질 수밖에 없으며, CSS를 얼마나 제대로 사용하는가에 따라 웹의 가치도 높아질 수 있습니다. CSS는 현재 CSS Level 1과 CSS Level 2를 지나 CSS Level2.1이 표준으로 자리 잡고 있으며, 모듈 방식으로 CSS가 업데이트 되고 있습니다.

최초의 웹은 연구자들을 위한 정보 교환 수단이었기 때문에 정형화되고 무미건조한 외형을 가지고 있었습니다. 그 후 웹의 확산과 상업화로 웹 사이트의 외형적 모습이 다양화되어야 할 필요성이 대두되었으며, 2000년대 초반까지 웹 사이트의 정보 구조를 해치면서 웹의 외형을 구현하는 방법을 사용했습니다. 웹의 외형을 구현하는 데 있어 CSS의 가장 큰 장점은 HTML이 가진 정보 구조를 해치지 않고 디자인을 적용할 수 있다는 것입니다.

# CSS의 과거와 현재

1996년 W3C의 주도하에 첫 번째 CSS 버전인 CSS Level 1이 발표되었습니다. 이 버전에는 텍스트와 관련된 속성, 여백 등이 포함되었고, 넷스케이프와 인터넷 익스플로러가 해당 버전을 지원하기 시작했습니다.

1998년에 CSS Level 2가 등장하면서 대부분의 웹 브라우저들이 CSS Level 2를 지원하기 시작했습니다. 그 후 CSS Level 2의 버그를 수정한 CSS Level 2.1이 2006년에 발표되면서 현재까지 표준으로 사용되고 있습니다.

CSS Level 3는 CSS Level 2.1과 달리 모든 명세가 포함된 버전이 아닌 모듈 단위로 개발되고 있으며, 표준화가 모듈 단위로 진행되고 있습니다. 이 중 몇몇 모듈은 현재 Recomendation(권고안) 단계에 있으며, Working Draft(초안) 단계에 머물러 있는 모듈도 있습니다.

CSS는 웹의 표현을 담당합니다. 이는 CSS Level 2와 CSS Level 3가 다르지 않습니다. 웹의 기능이 발전함에 따라 웹의 표현도 더 다양해지고 정교해질 필요가 있었습니다. 이러한 이유 때문에 CSS 표준 제정 방법에도 변화가 생겼습니다. CSS Level 2까지는 모든 기능에 대한 내용을 하나의 문서에 모두 정의하였지만 CSS Level 3부터는 기능에 따라 모듈로 분리하여 정의하고 있습니다.

CSS Level 3의 모듈에는 "text", "font", "color"와 같이 기존 CSS Level 2에 정의된 기능을 분리하여 더욱 구체화하고, 기능도 추가한 "transform", "transition", "animation" 등의 추가 모듈이 포함되어 있습니다.

CSS Level 3의 속성은 새롭게 추가된 것도 있지만, 기존 특정 브라우저에서 독자적으로 사용되던 속성이 표준 규격안으로 포함된 것도 있습니다. CSS가 모듈별로 개발됨에 따라 사용자 에이전트들이 필요한 모듈별로 구현하거나 업데이트를 할 수 있게 되었는데, 이를 통해 CSS Level 3의 지원에 빠르게 대응할 수 있을 것이라 생각합니다.

HTML이 진화함에 따라 CSS 역시 발전하고 있습니다. 웹이 HTML5로 인해 동적 애플리케이션화 되어가고, 웹을 표현할 수 있는 기기가 다양해짐에 따라 CSS 역시 CSS3로 표현의 범위를 넓혀 가고 있습니다.

CSS는 다른 W3C의 표준안과 동일하게 Draft(초안), Working Draft(작업 초안), Candidate Recommendation(권고 후보), Proposed Recommendation(최종 권고안), Recommendation(권고안) 순으로 만들어집니다. CSS3는 모듈별로 개발되고 있기 때문에 모듈별로 개발 진척 사항이 다르지만, 주요 모듈들은 Candidate Recommendation(권고 후보) 단계까지 진행된 상태입니다.

📌CSS 모듈

  • Selectors Level 3 : CSS 속성을 적용하기 위해 HTML 요소를 선택하는 방법에 대한 규정으로, CSS3에서는 이전 버전에서 제공하던 선택자 이외에 HTML의 정보 구조 관계를 탐색하여 요소를 선택하는 Structural Pseudo Selector가 다수 추가되었습니다.
  • CSS Color Module Level 3 : CSS에서 사용할 수 있는 색상에 관한 규정으로, CSS3에서는 색상의 투명도를 지정할 수 있는 RGBA 형식을 사용할 수 있게 되었습니다.
  • CSS Namespaces Module : CSS가 XML 형식에서 사용될 경우의 "네임 스페이스"를 지정하기 위한 규정입니다.
  • CSS Backgrounds and Borders Module Level 3 : CSS를 이용한 웹 디자인에서 사용 빈도가 높은 배경과 테두리에 대한 규정으로, 많은 기능이 추가되었습니다. 하나의 HTML 요소에 여러 개의 배경 이미지를 지정할 수 있으며, 요소의 테두리에도 이미지를 지정할 수 있습니다.
  • CSS Multi–Column Layout Module : 인쇄물의 다단 구성과 같은 형태의 레이아웃을 구현하기 위한 규정입니다.
  • Media Queries : Media Queries는 CSS의 media 속성의 확장 기능입니다. 기존의 media 속성에서는 웹을 표시하는 매체의 형식만을 구분하였지만, CSS3 추가된 기능으로 인해 매체의 특성에 따라서도 CSS 속성을 구분하여 적용할 수 있게 되었습니다.
  • CSS Mobile Profile 2.0 : 휴대 전화에 적용하기 위한 CSS의 하위 집합으로, CSS의 전체 스펙을 축소한 형태입니다.
  • CSS Marquee Module Level 3 : 움직이는 텍스트와 같이 사용자의 인터렉션에 독립적으로 흐르는 효과에 대한 규정입니다.
  • CSS3 Paged Media : 인쇄된 웹과 같이 페이지로 구분되는 미디어를 위한 규정입니다.
  • CSS Print Profile : 성능이 낮은 저가형 프린터에 적용하기 위한 CSS의 하위 집합으로, CSS의 전체 스펙을 축소한 형태입니다. CSS Values and Units : CSS에서 사용되는 값과 단위에 대한 규정입니다.
  • CSS Cascading and inheritance : CSS의 주요한 특징인 단계적 상속에 관한 규정입니다.
  • CSS Text Level 3 : 문자열에 관한 규정입니다.
  • CSS Writing Modes Module Level 3 : 문자에 따른 글쓰기 방향에 관한 규정입니다.
  • CSS3 Line Grid : 영문자의 베이스라인과 같은 문자열의 가이드 라인에 대한 규정입니다.
  • CSS3 Ruby Module : 한자나 일본어 등에서 독음 등을 표시하기 위한 작은 글자에 대한 규정입니다.
  • CSS Generated Content for Paged Media Module : 출력물과 같은 페이지로 나누어지는 매체에서 쪽 번호나 장번호를 생성하기 위한 규정입니다.
  • CSS Fonts Module Level 3 : 글자체에 관한 규정입니다. CSS3에서는 사용자 측에 미리 설치되어 있는 폰트뿐만 아니라 서버 측의 폰트도 지정할 수 있습니다.
  • CSS basic box model : CSS가 HTML 요소를 표시하는 기본 규칙인 Box Model에 관한 규정입니다.
  • CSS Template Layout Module : CSS 속성으로 레이아웃 템플릿을 작성하기 위한 규정입니다.
  • CSS Speech Module : 음성 합성 장치를 위한 볼륨, 음성, 속도, 피치 등을 제어할 수 있는 속성에 대한 규정입니다.
  • CSS3 Basic User Interface Module : form 요소의 서식과 같이 기본적으로 제공되는 사용자 환경에 대한 설정 방법의 규정입니다.
  • CSS Scoping : CSS 속성의 적용 범위에 대한 규정입니다.
  • CSS Grid Positioning Module Level 3 : Grid Layout에서의 요소 배치에 관한 사항에 대한 규정입니다.
  • CSS Grid Template Layout : 그리드 기반의 레이아웃을 구현하기 위한 속성에 대한 규정입니다. 기존 CSS에서 규정하던 position 속성 등과는 별도의 것입니다.
  • Flexible Box Layout Module : CSS3에 등장한 새로운 방식의 레이아웃을 구성하는 방법에 대한 규정으로, 박스의 배치 순서나 방향을 지정하는 방법에 대한 규정입니다.
  • CSS Image Values and Replaced Content Module Level 3 : 이미지나 대치되는 요소를 위한 속성들에 대한 규정입니다. 이 속성들은 CSS에서 사용되는 이미지 또는 대치되는 요소 자체에 대한 제어를 목적으로 합니다.
  • CSS 2D Transforms Module Level 3 : 2차원 형태 변환을 위한 규정입니다. 이 속성은 CSS 요소 박스의 확대, 축소, 회전 등의 변환을 적용합니다.
  • CSS 3D Transforms Module Level 3 : 3차원 형태 변환을 위한 규정입니다.
  • CSS Transitions Module Level 3 : CSS 속성 효과의 전환을 일정 시간 동안 부드럽게 진행하기 위한 규정입니다. Transition이 수행되기 위해서는 요소에 지정된 CSS의 속성 값이 변경되어야 하므로 특정한 트리거가 필요합니다.
  • CSS Animations Module Level 3 : CSS로 애니메이션을 구현하기 위한 속성에 관한 규정입니다. Transition도 애니메이션의 형태로 구현되지만, 이 속성은 애니메이션을 수행하기 위한 트리거가 특별히 필요없습니다.
  • CSS Style Attributes : HTML 등의 마크업 언어에 inline으로 CSS를 적용하기 위한 규정입니다.
  • CSSOM View Module : 스크롤된 거리나 화면 폭과 같은 화면이 보여지고 있는 형태에 대한 정보에 기계적으로 접근할 수 있는 CSSOM의 부분적인 규정입니다. CSS Extended Box Model : CSS의 기본 Box Model을 추가적으로 제어하기 위한 규정입니다.
  • CSSOM : CSS의 Object Model에 대한 규정으로, 스크립트나 API를 위한 정보를 제공합니다.
  • CSS3 Syntax : CSS를 작성하는 문법에 대한 규정입니다.
  • CSS Lists and Counters Module Level 3 : 리스트 요소를 위한 속성과 번호 지정을 위한 규정입니다.
  • CSS Tables : 테이블 요소를 위한 속성에 관련된 규정입니다.
  • The CSS ‘Reader’ Media Type : 화면 판독기와 같은 리더 기기를 위한 규정입니다.
  • CSS Positioning : 요소의 위치를 지정하는 속성에 관련된 규정입니다.
  • CSS3 Generated and Replaced Content Module : CSS로 내용 요소 또는 대치되는 요소를 생성하는 것에 대한 규정입니다.
  • CSS3 Line Layout : Inline Box에 대한 규정입니다.
  • CSS3 Hyperlink Presentation Module : 하이퍼링크가 표시되는 형태에 대한 내용을 다룹니다.
  • CSS Math : 수식을 작성하는 MathML의 스타일을 위한 규정입니다.
  • CSS Aural Style Sheets : 음성 장치를 위한 규정입니다.
  • CSS TV Profile 1.0 : TV에 적용하기 위한 CSS의 하위 집합으로, CSS의 전체 스펙을 축소한 형태입니다.
  • Behavioral Extensions to CSS : CSS를 확장하기 위한 동적 제어에 대한 규정입니다.
  • Cascading Style Sheets (CSS) : CSS에 대한 전반적 설명과 CSS3의 각 모듈에 대한 간략한 소개를 담은 문서입니다.

CSS3의 모듈들은 많은 것들이 작성 중입니다. 그러므로 위에 소개한 각 모듈은 다른 모듈과 통합되거나 삭제될 수 있으며, 필요에 따라 새로운 모듈이 추가될 수도 있습니다. 어떤 모듈은 CSS3 버전이 완료된 후에 다음 버전이 개발되고 있는가 하면, 어떤 모듈은 이전 버전에서의 개발이 더 이상 발전이 없는 것으로 보이는 것도 있으며, 새롭게 추가된 기능이지만 아직 콘셉트가 명확하지 않은 것도 있습니다.

이와 같이 모듈이 명확하게 정의되지 않았음에도 불구하고 각 모듈들을 간략하게 설명한 이유는 다양한 매체의 지원과 프로그램 인터페이스로서의 CSS가 개발되는 방향의 흐름을 보여준다고 생각하기 때문입니다. 또한 CSS3 역시 HTML5와 같이 표준 문서의 진척보다 웹 브라우저의 구현 속도가 더 빨라 보입니다.

웹 브라우저별 CSS3 지원 수준은 CSS3 Test 사이트를 통해 확인할 수 있습니다.

# CSS 사용의 의의

스타일 사용의 중요한 의의는 문서의 구조와 표현을 분리할 수 있다는 점입니다. 이는 구조와 표현을 분리함으로써 문서 구조의 수정 없이 스타일의 변경만으로 다양한 표현을 할 수 있다는 것을 의미하며, 웹 사이트에서도 동일한 장점을 가집니다.

웹 문서에 있어 구조와 표현의 분리는 워드프로세서 문서보다 중요할 수 있습니다. 그 이유 중 하나는 웹 문서가 기계적으로 해석될 수 있는 가능성이 높아지기 때문입니다. 기계적으로 해석되는 것은 CSS가 아니라 HTML이지만, 구조와 무관한 표현적 요소를 CSS로 분리함으로써 HTML은 간결해지고 더욱 구조화될 수 있기 때문입니다.

웹이 기계적으로 해석되는 것의 사례로는 Search Engine Optimization(SEO)을 들 수 있습니다.

웹 기술(HTML/CSS)은 온라인상의 문서를 작성하기 위한 규격이었지만, 기술의 발전하면서 점차 애플리케이션적인 성격도 포함하게 되었으며, 이에 따라 CSS는 단순히 정적 문서를 꾸미는 스타일 기능에서 머물지 않고 웹 애플리케이션의 User Interface를 구성하는 기술로 사용되고 있습니다. CSS3에서도 동적 표현 기능이 다수 추가되어 이러한 추세를 반영하고 있습니다.

# CSS 기초

CSS는 웹의 표현을 담당하는 언어로, 일반적인 컴퓨터 언어와는 다르게 다음과 같은 단순한 구조로 이루어져 있습니다. 하나의 규칙은 크게 선택자(selector)와 선언부(declaration block)로 이루어집니다. 그리고 선언부는 다시 속성(property)과 속성 값(value)으로 이루어지며, 선언부는 세미콜론(;)으로 속성과 속성 값을 구분하여 여러 개의 선언(declaration)을 지정할 수 있습니다.

여기서 잠깐

CSS 선언이 끝날 때 꼭 세미콜론(;)을 지정해야 하나요?

세미콜론(;)은 1개 이상의 속성값을 선언할 때 구분하기 위해 사용합니다. 하나의 속성만 선언하거나 마지막에 사용하는 경우에는 p {color: red}와 p {color: red; border:1px}처럼 생략할 수 있습니다. 그런데 마지막에도 세미콜론을 쓰는 이유는 스타일 속성을 추가 및 수정할 때 실수와 오류를 줄이고, 코드를 정렬할 때 가독성을 높이기 위해서 입니다. 이 부분은‘꼭 써야 한다, 안 써야 한다’라기보다는 작업할 때 먼저 규칙을 정해 통일성 있게 작성하는 것이 더 중요합니다.

📌CSS3의 웹 브라우저별 접두사(vendor prefix)

CSS3는 표준안이 아직 확정되지 않은 상태이기 때문에 최신 웹 브라우저들은 CSS3 속성을 실험적으로 제공하고 있습니다. 이를 위해 속성이나 속성 값 앞에 웹 브라우저별로 접두사(vendor prefix)를 제공합니다. 이 접두사의 경우 웹 브라우저별로 다르기 때문에 하나의 속성을 선언하기 위해서는 여러 번의 동일한 선언을 지정해야 합니다.

다음은 웹 브라우저별 접두사(vendor prefix) 입니다.

웹 브라우저 접두사(prefix)
파이어폭스(Firefox) –moz–
크롬, 사파리(Chrome, Safari) –webkit–
오페라(Opera) –o–
인터넷 익스플로러(Internet Explorer) –ms–

CSS3 속성을 선언하기 위해 사용해야 하는 접두사는 중복된 코드를 작성해야 하기 때문에 불편함을 느낄 수 있습니다. 이를 해결하기 위해서는 웹 브라우저별 접두사를 자동으로 생성해주는 CSS Prefix 사이트나 prefix-free를 이용하는 것이 좋습니다.

CSS는 속성 값을 지정할 때 여러 형식과 단위를 사용합니다. CSS3에서는 값과 단위에 대한 정의를 "CSS Values and Unite Module Level 3"로 개발되고 있습니다.

  • 문자열 타입 : "inherit"와 같이 CSS에서 미리 정의된 키워드나 제작자가 정의한 저작자 키워드 등 텍스트로 입력하는 속성 값이 이에 속합니다.
  • 숫자 타입 : 개수 비율들을 나타낼 수 있는 정수, 실수 값, 퍼센트 값이 이에 속합니다.
  • 길이 단위 타입 : 길이 단위 타입은 상대 길이 단위 값과 절대 길이 단위 값이 있습니다. 상대 길이 단위 값은 특정 길이에 비율로 정해지는 값으로, CSS3에서는 다음과 같은 단위 들이 정의되어 있습니다.

이 단위들은 기존 CSS2에서 음성 브라우저 등의 특수 경우를 상정하여 정의되었던 것이지만, CSS3에서는 표현 속성이 다양해짐에 따라 각도나 시간은 일반적으로 사용되는 값의 형식이 될 것입니다.

CSS3에서의 색상 단위는 좀 더 확대되었습니다. 특히, 투명 값(alpha)이나 색상, 채도, 명도 등을 지정할 수 있는 HSL 방식 그리고 currentcolor 키워드 등이 추가되었습니다. 다음은 CSS3에서 사용할 수 있는 색상 값입니다.

  • 색상 이름 : "red", "blue"와 같이 색 이름으로 지정하는 방법으로, CSS3에서는 147개의 색상 이름이 정의되어 있습니다.
  • transparent : 색상을 투명하게 지정합니다.
  • 16진수 RGB 색상 값 : 빛의 삼원색인 빨간색, 녹색, 파란색의 비율을 각각 두 자리 16진수로 지정하는 방법으로, 빨간색은 "#FF0000"으로 표시하며, "#F00"와 같이 각 색상을 한 자리로 단축하여 사용할 수도 있습니다.
  • 256단계 RGB 색상 값 : 빛의 삼원색인 빨간색, 녹색, 파란색의 비율을 각각 256단계의 숫자로 표시하는 방법으로, 빨간색은 "rgb(255,0,0)"으로 표시합니다.
  • 백분율(%) RGB 색상 값 : 256단계 RGB와 동일한 방법이나 색상의 비율을 0~100%의 값으로 지정합니다. 빨간색은"rgb(100%, 0%, 0%)"로 표시합니다.
  • RGBA 색상 값 : 빛의 삼원색인 빨간색, 녹색, 파란색과 투명도를 함께 지정하는 방법으로, CSS3에서 추가되었습니다. 투명도 값은 0~1 사이의 값을 가지며, "0"은 완전 투명하고 "1"은 불투명한 색상으로 표시됩니다.
  • HSL 색상 값 : 색상, 체도, 명도 값으로 색상을 지정하는 방법으로, CSS3에서 추가된 방식입니다.
  • HSLA 색상 값 : RGBA와 같이 HSL 색상 값과 투명도를 함께 지정하는 방식입니다.
  • currentcolor : currentcolor 키워드는 현재 요소 박스의 color 속성 값을 의미합니다. 예를 들어 border-color의 색상을 지정할 때 border-color : currentcolor;로 선언하면 테두리 색상이 글자 색상과 동일하게 적용됩니다.

📌CSS 적용하기

HTML 문서에 CSS를 적용하는 방법은 크게 CSS 파일을 외부에 생성하여 HTML 문서에 연결하는 External 방식, HTML 문서의 head 요소 내에서 style 요소를 선언하고 CSS 코드를 포함시키는 Embedded 방식, HTML 문서의 특정 요소 내에서 style 속성에 직접 CSS 코드를 지정하는 Inline 방식으로 나눌 수 있습니다.

다음은 HTML 문서에 CSS를 적용하는 3가지 방법을 정리한 것입니다.

 <meta charset="utf-8">

 <link rel="stylesheet" href="css/style.css">
 <style>
  @import url(css/style.css);
 </style>
</head>
<body>
 <h1>External 방식</h1>
 <p>&lt;link&gt; 요소를 사용하여 외부에 생성한 CSS 파일을 연결합니다.</p>

여기서 잠깐

하나의 CSS 파일에서 여러 개의 CSS 파일을 import할 경우 @charset 선언은 어떻게 지정해야 하나요?

하나의 CSS 파일에서 여러 개의 CSS 파일을 import하는 경우에는 가장 먼저 import할 CSS 파일에 @charset을 선언하거나 여러 개를 import할 CSS 파일에 @charset을 선언하면 됩니다. 여러 개의 CSS 파일을 사용해도 @charset은 한 번만 선언해야 합니다.
