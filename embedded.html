# 임베디드 관련 요소

# img 요소

웹 문서에 텍스트가 아닌 이미지 개체를 삽입할 때는 빈 요소인 img 요소를 이용해야 하며, img 요소의 경우 src 속성과 alt 속성은 필수 속성입니다. 그러나 이들 두 가지 속성 외에 이미지의 크기 정보를 지정하는 것이 좋습니다. 이미지의 크기 정보를 지정할 경우에는 width와 height 속성을 사용합니다.

특히 alt 속성은 이미지가 보이지 않는 환경에서 대체 텍스트를 제공하기 위한 목적으로 사용하며, 이미지와 동등한 정보를 제공해야 합니다. 만약 alt 속성으로 대체 텍스트를 지정하기 어려운 경우에는 longdesc 속성을 이용하여 이미지에 대한 대체 정보로 연결할 수 있습니다. 또 IR 기법과 같은 다양한 기술을 이용하여 이미지에 대한 대체 콘텐츠를 제공할 수 있습니다.

주의 사항

alt 속성을 지정했을 경우 인터넷 익스플로러 7 이하 버전에서 풍선 도움말(tooltip)로 나타나는 기능은 W3C의 권고를 MS가 오용하여 발생한 버그입니다. 따라서 웹 표준을 제대로 지원하는 웹 브라우저에서는 alt 속성을 지정해도 풍선 도움말이 나타나지 않습니다. 풍선 도움말로 표시하려면 title 속성을 활용해야 합니다.

이미지 요소에 alt 속성을 사용하여 이미지와 동등한 수준의 대체 텍스트를 제공했을 때 이미지가 보이지 않는 상황에서 대체 텍스트가 노출됩니다.

# iframe 요소

인라인 프레임을 이용하여 콘텐츠를 삽입할 경우에는 iframe 요소를 사용합니다. 인라인 프레임은 인터넷 익스플로러의 독자적인 요소였으나 HTML 4.0과 HTML 4.01 이후에 정식으로 지원하고 있습니다. 그러나 인라인 프레임은 XHTML 1.0 Strict와 XHTML 1.1에서는 사용할 수 없습니다. 인라인 프레임을 사용할 경우에는 웹 접근성을 고려하여 iframe 요소에 title 속성을 함께 제공하는 것이 바람직합니다.

title 속성에는 iframe의 콘텐츠가 어떤 내용을 담고 있는지를 설명해야 합니다. 이것은 시각 장애인이 스크린 리더를 이용하여 콘텐츠에 접근할 경우 iframe의 콘텐츠 내용을 파악하는 데 도움을 줍니다. 서버 측의 데이터를 교환할 목적을 가진 빈 iframe이라 하더라도 반드시 title 속성으로 설명을 제공해야 합니다.

👍iframe이 지원되지 않는 환경까지 고려하기

프레임이 지원되지 않는 경우에는 iframe 요소의 내용이 그대로 표시될 수 있으므로 iframe이 지원되지 않는 환경까지 고려하여 대체 콘텐츠를 입력해야 합니다. title 속성은 iframe 요소에 담긴 콘텐츠를 잘 나타낼 수 있는 내용으로 지정합니다.



 
 


<iframe src="ad_banner.html" name="ad" frameborder="0" 
width="350" height="50" scrolling="no" title="배너 광고">
 <p>프레임이 표시되지 않는 분은 
  <a href=" ad_banner.html">배너 광고 바로가기</a>를 클릭해 주세요.</p>
</iframe>

👎iframe에서 잘못된 방법으로 이미지를 불러오는 경우

이미지가 나타나도록 이미지 파일을 src 속성으로 불러오는 것은 잘못된 방법입니다. 새로운 문서를 생성한 후 이미지를 삽입하고 해당 문서를 불러와야 합니다.


 
 


 
 



- 이미지를 직접 iframe 내에서 불러오는 경우  

<iframe src="girl_picture.jpg" name="girl" frameborder="0" 
width="400" height="250" scrolling="no" title="소녀시대 이미지"></iframe> 

- 이미지는 HTML 문서에 포함하고 해당 HTML 문서를 iframe 내에서 불러오는 경우  

<iframe src="girl_picture.html" name="girl" frameborder="0" 
width="400" height="250" scrolling="no" title="소녀시대 이미지"></iframe>

# video, audio, source, track 요소

video, audio, source, track 요소는 HTML5에 새롭게 도입된 요소로, 동영상 및 오디오 콘텐츠를 삽입하고자 할 때 사용합니다. 동영상 및 오디오 콘텐츠의 경우 기존 HTML에서는 플러그인에 의존적이었지만, video 요소와 audio 요소의 등장으로 더 이상 플러그인에 의존하지 않고 동영상 및 오디오 콘텐츠를 제공할 수 있게 되었습니다. 또한 플러그인에 의존적이지 않기 때문에 CSS를 활용하여 디자인을 적용할 수 있고, 자바스크립트로 동영상 및 오디오 등을 제어할 수 있게 되었습니다.

video 및 audio 요소를 사용하여 동영상 및 오디오 콘텐츠를 삽입하는 경우, 웹 브라우저가 지원하는 동영상 및 오디오 형식이 통일되어 있지 않습니다. 그러나 2019년 3월 현재 기준으로 video 요소는 MPEG-4/H.264 video 포맷이 audio 요소는 MP3 audio 포맷이 Opera Mini를 제외한 거의 모든 웹브라우저에서 지원하고 있습니다.

video 요소나 audio 요소에는 다양한 속성을 지정할 수 있습니다. 먼저 autoplay 속성은 자동 재생 기능을 담당합니다. poster 속성은 동영상 콘텐츠가 재생되기 전에 미리 표시하고 싶은 이미지를 삽입하고자 할 때 사용합니다. controls 속성은 동영상이나 오디오의 재생, 정지 등의 인터페이스를 지정할 수 있으며, loop 속성은 재생이 끝난 후 다시 재생할 것인지를 결정하는 논리 속성입니다.

다음은 video 및 audio 요소에 사용할 수 있는 속성입니다.

  • src : 소스 파일
  • autoplay : 자동 재생
  • controls : 재생이나 정지 등의 인터페이스
  • loop : 재생 횟수
  • preload : 웹 문서 로드 시 재생할 미디어를 미리 로드
  • poster : 동영상의 포스터 이미지
  • mediagroup : 여러 개의 멀티미디어 콘텐츠 연결
  • muted : 음소거
  • crossorigin : CORS(Cross–Origin Resource Sharing) 자격 증명
  • width : 가로 크기
  • height : 세로 크기

track 요소는 자막이나 원고 등을 제공하기 위한 요소로, kind 속성에 "subtitles", "captions", "descriptions", "metadata" 중 하나를 명시하고 해당 자막 파일은 src 속성을 통해 제공할 수 있습니다.













 







 











<!DOCTYPE html>
<html lang="ko">

<head>
 <meta charset="utf-8">
 <title>video, audio, source, track 요소</title>
</head>
<body>
 <section>
  <h1>동영상 및 오디오</h1>
  <video width="640" height="360" controls="controls" 
         poster="images/chrome.jpg">
   <source src="media/chrome.mp4" type="video/mp4">
   <source src="media/chrome.webm" type="video/webm">
   <source src="media/chrome.ogv" type="video/ogg">
   <track label="English subtitles" kind="subtitles" srclang="en" 
          src="chrome_subtitles.vtt" default>
   <p>동영상 파일 : <a href="media/chrome.mp4">크롬 소개 동영상</a></p>
  </video>
  <audio controls="controls" preload="auto">
   <source src="media/interview.mp3" type="audio/mp3">
   <source src="media/interview.ogg" type="audio/ogg">
   <source src="media/interview.wav" type="audio/wav">
   <source src="media/interview.m4a" type="audio/mp4">
   <track label="Korean subtitles" kind="subtitles" srclang="ko" 
   src="interview_subtitles.vtt" default>
   <p>음성 파일 : <a href="media/interview.mp3">인터뷰</a></p>
  </audio>
 </section>
</body>
</html>

용어 설명

코덱(Codec)

코덱은 ‘compress(압축)’과 ‘decompress(비압축)’의 합성어입니다. 일반적으로 동영상이나 음성 콘텐츠 등은 압축되어 있는 상태이며, 이렇게 압축된 것을 화면으로 보거나 음성으로 들을 수 있도록 하기 위해서는 다시 압축을 풀어주어야 하는데, 이때 필요한 것이 바로 코덱입니다. HTML5에서는 가히 코덱 전쟁이라 불릴 만큼 H.264와 WebM이 경쟁하고 있으며, 이에 Ogg Theora가 가세하고 있는 형국입니다. 아직은 video, audio 요소에 코덱 등의 문제가 있지만 미래에는 이러한 부분이 해결되어 플러그인 없이도 웹 브라우저에서 동영상이나 음성 등을 자유롭게 재생할 수 있을 것으로 예상됩니다.
