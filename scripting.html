# script 요소

HTML 문서를 제작할 때 자바스크립트의 역할은 매우 중요합니다. 이러한 자바스크립트 코드를 직접 문서에 삽입하거나 외부에 별도로 작성한 자바스크립트 파일을 연결하고자 할 경우 script 요소를 사용할 수 있습니다.

또한 HTML5에서는 script 요소에 기존에 제공하던 src와 type 속성 이외에 async와 defer 속성이 추가되었습니다.

웹브라우저는 웹 문서에 자바스크립트 파일이 지정된 경우 해당 파일이 모두 다운로드 될 때까지 HTML 파싱을 중단합니다. 그러나 async와 defer를 지정하면 자바스크립트 파일 다운로드와 HTML 파싱을 동시에 진행하게 됩니다. 이때 async의 경우 자바스크립트 파일의 다운로드가 완료되면 바로 실행을 하면서 HTML 파싱을 중단하고 defer는 자바스크립트 파일의 다운로드가 완료되었더라도 바로 실행하지 않고 HTML 파싱이 끝난 후 실행한다는 차이점이 있습니다.






 
 
 
 
 
 














<!DOCTYPE html>
<html lang="ko-KR">
<head>
 <meta charset="utf-8">
 <title>script 요소</title>
 <script src="js/hello.js" defer></script>
 <script>
  window.onload = function(){
  hello();
 }
 </script>
</head>
<body>
 <h1>async와 defer 속성</h1>
 <p>웹 문서에서 자바스크립트 파일을 실행할 경우 해당 파일이 모두 다운로드 될 때까지
  HTML 파싱을 중단합니다. 그러나 async와 defer를 지정하면 자바스크립트 파일 다운로드와 
  HTML 파싱을 동시에 진행합니다.</p>
 <p>이때 async와 defer의 차이점은 async의 경우 자바스크립트 파일의 다운로드가
  완료되면 바로 실행을 하면서 HTML 파싱을 중단하고 defer의 경우 자바스크립트 파일의
  다운로드가 완료되었더라도 바로 실행하지 않고 HTMl 파싱이 끝난 후 실행한다는
  차이점이 있습니다.
 </p>
</body>
</html>

# noscript 요소

noscript 요소는 웹 브라우저가 script 요소를 지원하지 않을 경우에 보여주고자 하는 대체 콘텐츠를 지정할 때 사용할 수 있습니다.

<!DOCTYPE html>
<html lang="ko-KR">
<head>
 <meta charset="utf-8">
 <title>noscript 요소</title>
 <script type="text/Javascript">
  window.open("https://seulbinim.github.io/")
 </script>
</head>
<body>
 …
 <noscript>
  <a href="https://seulbinim.github.io/" target="_blank">웹 카페 바로가기</a>
 </noscript>
</body>
</html>
