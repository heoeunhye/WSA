# 폼 관련 요소

이제까지 살펴보았던 다양한 요소들과 달리 폼 요소는 웹 문서가 서로 상호작용을 할 수 있도록 하는 역할을 담당합니다. 사용자가 자료를 검색하기 위한 검색어 입력 상자나 여러 가지 항목에서 원하는 값을 선택할 수 있도록 제공하는 라디오 버튼 또는 체크 박스, 목록 형태로 펼쳐지는 목록 상자(combo box), 서버로 자료를 전송하기 위한 전송 버튼 등이 이에 해당됩니다.

웹 문서는 서버에 미리 지정된 위치로 사용자가 입력하거나 선택한 값을 전송하는데, 이때 서버에서는 사용자가 송신한 자료를 데이터베이스에 저장하거나 처리합니다. 서버 처리와 관련된 프로그램으로는 CGI(common Gateway Interface)나 PHP(Personal Hypertext Preprocessor), JSP(JavaServer Pages), ASP(Active Server Pages) 등이 있습니다.

이러한 서버 처리 프로그램들은 하나 이상의 작은 내장프로그램(스크립트)을 갖고있는 HTML 페이지를 사용자가 볼 수 있도록 도와줍니다.

다음 그림은 폼 컨트롤 요소에 사용자가 인증 정보(아이디, 비밀번호)를 입력한 후, 서버로 정보를 전송하면 서버는 요구받은 인증 정보를 데이터베이스에서 추출하여 인증 결과를 알려 주는 과정을 나타낸 것입니다.

# form 요소

폼의 범위를 정의할 때는 form 요소를 사용합니다. form 요소의 속성으로는 폼의 내용을 처리하는 서버의 URI를 지정하기 위한 action과 폼의 내용을 처리하는 방법인 method가 있고, 사용 형식은 다음과 같습니다.

<form action="서버 URI" method="GET 또는 POST">
  폼의 내용
</form>
1
2
3
  • action=“URI” : 폼의 내용을 처리하기 위한 서버의 URI(필수)
  • method=“GET 또는 POST”
    • GET : action 속성에 지정한 URI에 폼의 내용을 추가하여 서버에 송신하는 방식으로, form을 지정할 때 기본값으로 설정되어 있습니다. GET 방식은 데이터의 길이가 짧을 경우에 적합하며, POST 방식보다 처리 속도가 약간 빠릅니다. GET 방식의 경우 웹 브라우저의 주소 창에 변수값이 노출되기 때문에 보안상 중요한 데이터라면 POST 방식을 사용하는 것이 바람직합니다.
    • POST : 폼의 본문으로 송신하는 방식으로, 데이터의 길이가 긴 경우에 적합하며, 보안 문제 때문에 데이터값을 노출시킬 수 없을 때 사용합니다.
  • accept–charset : 폼 전송에 사용할 문자 인코딩 지정(문자 인코딩 값을 둘 이상 지정 가능하며, 이때 문자 인코딩 값을 공백으로 구분하여 선언)
  • autocomplete : 자동 완성 기능의 활성화 여부(on, off 값 지정)
  • enctype : 폼 전송 데이터의 MIME 타입
  • name–폼의 이름
  • novalidate–폼 전송 시 입력 데이터 값의 검증을 수행하지 않음(논리 속성)
  • target–폼의 전송 값을 처리할 대상









 
 
 









 




<!DOCTYPE html>
<html lang="ko">
<head>
 <meta charset="utf-8">
 <title>form 요소</title>
</head>
<body>
 <section>
  <h1>폼 삽입</h1>
  <form action="login.php" method="post" name="memberLogin"
        accept-charset="utf-8" autocomplete="on"
        target="_blank" novalidate="novalidate">
  <p>
   <label for="userId">아이디</label>
   <input type="text" id="userId">
  </p>
  <p>
   <label for="userPw">비밀번호</label> 
   <input type="password" id="userPw">
  </p>
  <p><input type="submit" value="로그인"></p>
  </form>
 </section> 
</body>
</html>

# fieldset, legend 요소

여러 개의 폼 요소를 그룹화하여 좀 더 구조적으로 만들려면 fieldset 요소를 사용해야 합니다. fieldset 요소를 이용하면 fieldset의 주위에 border가 생성되어 해당 그룹 안에 있는 콘텐츠가 폼과 관련된 서식임을 쉽게 구별할 수 있습니다. 또 legned 요소로 fieldset에 포함 된 폼 서식이 어떠한 성격인지 제목 형식으로 알려 줄 수 있으며, legend 요소는 fieldset 요소의 자식 요소로 한 번만 지정할 수 있습니다.

다만 legend 요소의 경우 크로스 브라우징 관점에서 웹 브라우저 스타일이 다르게 적용되어 있으므로 CSS를 이용한 디자인 작업에는 어려움이 있을 수 있습니다.

fieldset과 legend 요소의 사용 형식은 다음과 같습니다.


 
 

 


<form action="member.asp" method="POST">
 <fieldset>
  <legend>폼 요소의 제목</legend>
   … 생략 …
 </fieldset>
</form>

HTML5에서 fieldset 요소에 새롭게 추가된 속성인 form 속성을 사용하면 form 요소 안에 반드시 fieldset 요소를 포함시키지 않아도 해당 form 요소와 연결할 수 있습니다. 연결하는 방법은 form 요소에는 id 값을 할당하고, fieldset 요소에는 속성으로 form을 지정하며, 해당 속성 값으로 연결하고자 하는 form 요소의 id 값을 지정하면 됩니다.

 



 












<form id="memberLogin" action="login.php" method="post" 
      name="memberLogin" accept-charset="utf-8" target="_blank" 
      autocomplete="on" novalidate="novalidate">
</form>
<fieldset form="memberLogin">
 <legend>회원 로그인</legend>
 <p>
  <label for="userId">아이디</label>
  <input type="text" id="userId">
 </p>

  <input type="password" id="userPw">

</fieldset>

# label 요소

폼을 구조화하고 접근성을 높일 수 있는 요소 중에는 label 요소가 있습니다. label 요소는 각 폼 서식의 연관 관계와 설명을 추가하는 역할을 담당합니다. 웹표준을 지원하는 최신 웹 브라우저의 경우 레이블만 선택해도 폼 서식을 선택할 수 있으며, 스크린 리더의 경우 폼 서식이 레이블과 인접하지 않는 경우에도 인식할 수 있도록 지원합니다.

label 요소는 웹접근성 측면에서도 중요한 요소로, 다양한 폼 서식의 설명을 의미합니다. 가령 아이디 입력 상자나 비밀번호 입력 상자의 경우, 기계가 해당 입력 상자가 어떤 데이터를 입력하는 영역인지 알 수 없기 때문에 label 요소로 연관 관계의 의미를 설명할 수 있습니다. 이렇게 폼 서식의 연관성을 명확하게 지정하면 웹표준을 지원하는 최신 웹 브라우저의 경우 레이블만 선택해도 폼 서식을 선택할 수 있으며, 스크린 리더의 경우 폼 서식이 레이블과 인접하지 않는 경우에도 인식할 수 있게됩니다.

label 요소로 연관 관계를 설명할 수 있는 폼 서식으로는 button, input, keygen, meter, output, progress, select,textarea 요소를 들 수 있습니다.

label 요소를 사용하여 폼 서식과의 연관 관계를 설명할 때는 for 속성을 사용할 수 있습니다. for 속성에서는 연결하고자 하는 폼 서식의 id 속성 값을 지정해야 하며, fieldset 요소와 마찬가지로 새로 추가된 form 속성도 사용할 수 있습니다.

이러한 label 요소의 사용에는 명시적인 방법과 암묵적인 방법의 두 가지가 있는데, 암묵적인 방법보다는 명시적인 방법이 더 권장됩니다. 폼 서식을 설명할 때 label 요소를 사용하는 것이 어려울 경우에는 title 속성을 활용할 수도 있습니다.






 
 








 
 
 




- 명시적 label(id와 for 속성 연결하기)
<form action="member.asp" method="post">
 <fieldset>
  <legend>회원가입 정보</legend>
  <p>
   <label for="userName">이름</label>
   <input type="text" id="userName" name="name" value="value" />
  </p>
 </fieldset>
</form>
- 암묵적 label(label 요소에 폼 컨트롤 포함시키기)
<form action="member.asp" method="post">
 <fieldset>
  <legend>회원가입 정보</legend>
  <p>
   <label>이름
    <input type="text" id="userName" name="name" value="value" />
   </label>
  </p>
 </fieldset>
</form>

# input 요소

input 요소는 여러 가지 폼 서식을 마크업할 때 사용합니다. 특히 HTML5에서는 기존에 제공하던 10가지 type 이외에 search, tel, url 등 다수의 type이 새롭게 추가되었습니다. 또한 type 속성 이외에 autocomplete 속성, list 속성, required 속성, multiple 속성, maxlength 속성, pattern 속성, max 속성, min 속성, step 속성, placeholder 속성, autofocus 속성 등 다수의 신규 속성도 추가되어 성능이 한층 더 강화되었습니다.

그러나 많은 신규 기능이 아직 모든 웹 브라우저에서 지원되고 있지 않으므로 사용 시 웹 브라우저에서 해당 기능을 제공하는지 확인해야 합니다.

input 요소는 type 속성 값에 따라 폼 서식의 종류가 결정됩니다. 지금부터 input 요소에서 사용할 수 있는 다양한 속성 값에 대해 살펴보겠습니다.

📌type="text"

input 요소에서 type 속성에 사용하는 "text" 타입은 한 줄 글상자를 나타낼 때 사용합니다.

<label for="userId">비밀번호</label>
<input type="text" id="usertext" name="userPw">
1
2

📌type="password"

input 요소에서 type 속성에 사용하는 "password" 타입은 한 줄 글상자와 렌더링 결과가 동일하지만, 입력 값을 화면에 출력하지 않는 형태로 나타내기 때문에 주로 클라이언트 환경에서 해당 값을 보이지 않도록 할 때 사용합니다.

<label for="userPw">비밀번호</label>
<input type="password" id="userPw" name="userPw">
1
2

📌type="search"

"search" 타입은 한 줄 글상자인 "text" 타입과 별 차이가 없습니다. 다만 줄바꿈을 포함한 문자열 값이 입력될 경우 줄바꿈을 제거합니다. "search" 타입의 원래 목적은 "text" 타입과 구별하여 표시하기 위해서입니다. 이를 반영하고 있는 경우를 예로 들면 Mac OS 환경하의 사파리 등과 같은 웹 브라우저에서 "search" 타입의 렌더링 결과를 살펴보면 "text" 타입과 달리 해당 글상자가 둥근 모서리 형태로 보이는 것을 알 수 있습니다.

<label for="keyword">검색</label>
<input type="search" id="keyword" name="keyword">
1
2

📌type="tel"

"tel" 타입은 "text" 타입과 동일하며, 차이점은 입력 값이 전화번호를 위한 것임을 의미한다는 것입니다. 또한 줄바꿈을 포함한 문자열 값이 입력될 경우 줄바꿈을 제거합니다. "tel" 타입을 사용하면 모바일 등의 디바이스에서 해당 서식에 값을 입력할 때 숫자 입력 모드로 제공되어 사용성이 향상됩니다.

<label for="userTel">전화번호</label>
<input type="tel" id="userTel" name="userTel">
1
2

📌type="url"

"url" 타입은 한 줄 글상자와 동일하며, 입력 값이 URL을 위한 것임을 의미합니다. "url" 타입에 줄바꿈을 포함한 문자열이 들어올 경우에는 웹 브라우저가 이를 제거합니다. "url" 타입에 입력할 수 있는 값은 절대 경로 URL만 가능합니다. 만약 부적절한 URL을 입력할 경우에는 폼 전송 시 에러 메시지를 출력합니다. "url" 타입의 경우 모바일 디바이스에서 영문 및 숫자 입력 모드로 변경됩니다.

<label for="userHomepage">홈페이지</label>
<input type="url" id="userHomepage" name="userHomepage">
1
2

📌type="email"

"email" 타입은 한 줄 글상자와 동일하며, 입력 값이 이메일을 입력하기 위한 것임을 의미합니다. email 타입에 입력할 수 있는 값은 이메일 주소만 가능합니다. 만약 부적절한 이메일 주소가 입력될 경우 폼 전송 시 에러 메시지를 출력합니다. "email" 타입의 경우 모바일 디바이스에서는 영문 및 숫자 입력 모드로 변경됩니다.

<label for="userEmail">이메일 주소</label>
<input type="email" id="userEmail" name="userEmail">
1
2

📌type="date"

"date" 타입은 날짜를 입력하고자 할 때 사용합니다. "date" 타입을 사용하면 웹 브라우저에서 날짜를 입력할 수 있는 입력 폼을 함께 제공합니다. 그러나 아직 날짜 입력 폼을 제공하지 않는 웹 브라우저도 있습니다. "date" 타입의 경우, "min" 속성과 "max" 속성을 사용하여 입력 날짜의 범위를 지정할 수도 있습니다.

이때 min="2019-10-01" max="2019-10-31"라고 입력하면, 입력 날짜는 2019년 10월 1일부터2019년 10월 31일까지로 제한됩니다. 또한 "step" 속성을 사용하면 최소 단위를 변경할 수도 있습니다. 날짜를 입력하는 최소 단위는 1일로 설정되어 있지만, 7일 단위로 입력하고 싶다면 step="7"로 지정하면 됩니다.

<label for="reservationDate">예약가능 일</label>
<input type="date" min="2013-10-01" max="2014-12-31"
       step="7" id="reservationDate" name="reservationDate">

# button 요소

button 요소는 버튼 서식을 삽입할 때 사용합니다. button 요소에 type 속성을 지정하지 않으면 해당 버튼이 전송 버튼이 됩니다. button 요소에 지정할 수 있는 type 속성 값은 "submit", "reset", "button" 등이며, input 요소의 "submit", "reset", "button"과 기능이 동일합니다.

웹접근성을 위해 온라인 서식에 적절한 레이블을 제공하라는 부분이 있는데, button 요소에도 적용해야 하나요?

button 요소의 경우 스스로 어떤 콘텐츠인지를 알려 주기 때문에 레이블이 필요 없습니다. 이 밖에도 데이터 전송에사용하는 버튼은 value 속성이 레이블의 역할을 대신합니다. 이를 정리하면 다음과 같습니다.

  • 데이터 전송 및 취소 버튼(input type="submit", input type="reset") : value 속성 대체
  • 이미지 버튼(input type="image") : alt 속성 대체
  • button 요소(button, input type="button") : 콘텐츠 자체를 요소로 가짐(텍스트 또는 이미지의 alt 속성)
  • 숨김 input(input type="hidden")

그러므로 다음과 같이 이미지 버튼에 alt를 적용하거나, 레이블을 제공하는 것은 잘못된 방식입니다.

<label for="btnSearch">검색</label> <input type="image" id="btnSearch" alt="검색">

<input type="image" id="ok" alt="확인" />
<input type="button" name="idCheck" value="아이디 중복 확인" />
<input type="submit" name="cmdPoll" value="설문 완료" />
<button name="btnbuy">상품 구매! BUY!</button>
<button name="btngallery"><img src="btn_gallery.gif" alt="갤러리"></button>

# select, optgroup, option

select 요소는 항목을 선택할 수 있는 폼 서식을 마크업할 때 사용합니다. select 요소에서 지원하는 size 속성은 한 번에 표시할 항목의 수를 의미하며, multiple 속성은 다중 선택을 허용할 것인지의 여부를 결정하는 논리 속성입니다.

optgroup 요소는 select 요소 안에서 사용할 수 있는 그룹 요소로, 각 항목을 의미하는 option 요소를 그룹화하여 지정할 때 사용합니다. optgroup 요소의 경우, label 속성을 사용하여 해당 그룹의 역할을 이해할 수 있도록 제공할 수 있습니다.

파일 첨부 폼 요소의 디자인을 변경할 수 있나요?

전면적인 디자인을 변경하는 것은 불가능하고, 일부 영역에 CSS를 이용해서 border 및 배경색을 바꾸는 정도는 가능합니다. 간혹 자바스크립트나 다른 트릭을 이용해서 파일 첨부 폼 요소의 전면적인 디자인을 강제적으로 변경할 수 있지만, 가급적이면 사용자의 경험을 고려하여 입력 폼의 형태를 변형하지 않는 것이 좋습니다.

# datalist

datalist 요소는 HTML5에 새롭게 추가된 요소로, 텍스트 상자의 입력 값의 후보 목록을 그룹화할 때 사용합니다. 이러한 기능은 기존 HTML4.01이나 XHTML1.0에서 자바스크립트 등을 사용하여 구현했지만, 이제부터는 자바스크립트 없이 이러한 구현이 가능하도록 마크업상에서 지원합니다.

# textarea

여러 줄로 된 텍스트를 입력받을 때는 textarea 요소를 이용합니다. textarea 요소는 input 요소를 이용한 한 줄 글상자와 달리 textarea 요소 안에서 입력한 텍스트의 내용이 길어지면 임의의 줄로 바뀝니다. 이때 입력받을 수 있는 텍스트 수는 제한이 없습니다. HTML5에서 textarea 요소에 새롭게 추가된 속성으로는 placeholder, required 속성을 들 수 있습니다.

여러 줄 글상자의 영역 크기는 cols, rows 속성을 이용하여 지정할 수 있습니다. 예를 들어 cols=“60”row=“10”으로 지정하면 영문 A의 경우 A가 가로로 60개 들어가며, 10줄로 보이는 영역을 차지합니다. 단, 이 값은 웹 브라우저의 설정이나 글꼴의 크기값에 영향을 받으므로 환경에 따라 크기가 다르게 보일 수 있습니다. 따라서 정확하게 정렬하려면 CSS의 width, height 속성을 이용하여 영역의 크기를 지정합니다.

요소의 wrap 속성은 textarea 요소 안에서 자동 줄바꿈 처리된 텍스트에 줄바꿈 명령을 넣을 것인지의 여부를 결정하는 속성으로, 기본 값은 soft입니다. soft는 줄바꿈을 넣지 않겠다는 의미로, hard는 줄바꿈을 넣겠다는 의미로 사용합니다.

textarea 요소와 iframe 요소의 올바르지 못한 사용법

textarea 요소와 iframe 요소는 가로 세로 스크롤이 생기는 점을 이용하여 콘텐츠를 표현하는 데 사용하는 경우가 많습니다. 회원 약관의 콘텐츠를 textarea 요소를 사용하여 웹 문서의 일부분에서 스크롤이 생기도록 하는 것이 대표적인 예이지만, 이는 올바른 사용 방법이 아닙니다. 이와 같이 콘텐츠를 구성할 때 특정 영역을 지정하거나, 스크롤 효과가 필요한 경 우에는 CSS의 overflow 속성을 이용하면 됩니다.

# keygen 요소

keygen 요소는 폼을 전송할 때 키를 생성하는 서식으로, 전자 인증에 주로 사용합니다. keygen 요소를 사용하면 폼이 전송된 후에 비밀 키와 공개 키를 생성하여 비밀 키는 웹 브라우저에 저장하고, 공개 키는 서버에 전송합니다. HTML5에서 keygen 요소에 지원하는 keytype은 rsa만 가능합니다.

keygen 요소를 사용하면 웹 브라우저에서 RSA 전자 서명 알고리즘(md5withRSA-Encryption)에 사용할 암호화 비트 수를 선택할 수 있습니다.

   <label for="comment">보안등급</label><br>
   <keygen name="key" type="rsa">
  </form>
 </section>

# meter 요소

meter 요소는 progress 요소와 혼동할 수 있는 요소로, 작업의 진행 상태가 아닌 한정된 범위의 값을 나타내고자 할 때 사용합니다. 한정된 범위의 값을 의미하는 meter 요소를 사용하여 마크업할 때는 max 속성의 경우 최대값을, min 속성의 경우 최소값을 지정할 수 있습니다.

다음은 meter 요소에서 사용할 수 있는 속성입니다.

  • value : 값
  • min : 최소값
  • max : 최대값
  • low : 낮은 영역의 상한 값으로 지정하지 않으면 min 속성 값과 같은 값으로 간주
  • high : 높은 영역의 상한 값으로 지정하지 않으면 max 속성 값과 같은 값으로 간주
  • optimum : 최적 값으로 지정하지 않으면 max 속성과 min 속성의 중간 값으로 간주
