# 제목 관련 요소

# 제목 요소(h1 ~ h6)

제목은 콘텐츠의 내용에 따라 적절하게 지정해야 합니다. 이때 대제목, 중제목, 소제목 등으로 작성하며, 제목 요소는 h1, h2, h3, h4, h5, h6의 여섯 가지를 사용할 수 있습니다. 제목 요소는 블록 요소를 포함할 수 없으며, 인라인 요소와 텍스트만을 포함할 수 있습니다. 특히 잘 구성된 헤딩은 기기에서의 정보 인식성을 높일 수 있습니다.

예를 들어 문서의 구조 정보와 콘텐츠를 나타내는 제목을 가지고 있는 제목은 검색엔진 로봇이 중요한 정보로 인식하므로 검색엔진 최적화(SEO) 관점에서 바람직합니다. 또 시각 장애인의 경우 스크린 리더에서 웹 페이지의 제목 구조를 파악하여 제목 사이의 이동 기능을 제공하므로, 웹 페이지를 빠르게 탐색 할 수 있도록 도와줍니다.

여기서 잠깐

스크린 리더에서 제목(heading) 요소를 사용하면 어떻게 읽어 주나요?

국산 스크린 리더인 센스리더를 사용할 때 + 을 누르면 헤딩 간 이동을 할 수 있으며, 이전 헤딩으로 이동하려면 + + 을 누르면 됩니다. 이 경우 h1은‘헤딩1’, h2는‘헤딩2’로 읽는데, <h2>홍보배너</h2>는‘홍보배너 헤딩2’라고 읽고, 이미지를 사용한 <h1><img src=“subject_web.gif”alt=“웹 접근성이란”/></h2>은‘헤딩1 웹 접근성이란 이미지’라고 읽습니다.

이때 텍스트와 이미지를 제목 요소의 콘텐츠로 사용한 경우에는 읽어 주는 순서가 다릅니다. 외국에서 사용하는 스크린 리더 중 하나인 JAWS의 경우도 이와 같은데, 작업할 때 같은 레벨의 h 요소를 텍스트, 이미지 형식에 맞춰 주면 시각장애인이 스크린 리더로 웹을 탐색할 때 도움이 됩니다.

제목 관련 요소를 이용하여 마크업할 때는 h1 다음에 h2, h3순으로 나와야 계층 구조가 어긋나지 않으며 바로 h3으로 건너 뛰는 구조를 가질 경우에는 논리적인 웹 문서를 제작할 수 없으므로 주의해야 합니다.

👍콘텐츠의 의미를 전달하는 제목

메인 콘텐츠들의 디자인을 살펴보면 제목에 해당되는 디자인 요소가 있거나 없는 경우를 볼 수 있습니다. 메인 사이트의 콘텐츠 중 일부분인 아래 그림을 보면 뉴스, 사고 현황, 위치 찾기 서비스 등의 콘텐츠는 제목(h 요소)이 디자인으로 함께 표현되어 있습니다. 테두리에 들어가 있는 콘텐츠를 살펴보면 위쪽에 있는 배너 모음 콘텐츠와 아래쪽에 있는 3개의 배너에 대한 제목 디자인이 없는 것을 알 수 있습니다. .

눈으로 화면을 직접 보면 사이트 배너 모음인지, 회사 윤리 경영 방침에 대한 내용인지를 알 수 있지만 콘텐츠 제목 없이‘생활 안전 주부 봉사대 모집’이라는 텍스트 하나만으로는 공지사항인지, 알림 콘텐츠인지, 그리고 그 다음에는 어떤 정보가 나올 것인지를 예측하기 어렵습니다.

디자인상 생략된 제목 요소를 콘텐츠의 특성에 맞게 마크업하면 순차적으로 정보에 접근하는 시각 장애인에게 도움이 됩니다. 화면에 제목 요소가 표시되어 디자인상 문제가 되는 부분은 CSS를 이용하여 제목을 숨길 수 있습니다.

👎레이아웃 구조를 위한 제목 사용

간혹 웹 사이트의 레이아웃 정보를 헤딩으로 표현하고, 헤딩을 CSS를 이용하여 숨기는 경우가 있습니다. 다음과 같이 <h2>상단영역</h2>이라고 적은‘상단영역’은 무의미한 제목이 됩니다.








 





 


 
 









<html lang="ko-KR">
<head>
 <meta charset="UTF-8">
 <title>레이아웃 구조를 위한 제목 사용</title>
</head>
<body>
 <h1>생활안전공사</h1>
 <h2>상단영역</h2>
 <h3>주요메뉴</h3>
 <ul>
  <li>회사소개</li>
  <li>메뉴소개</li>
 </ul>
 <h2>중앙콘텐츠영역</h2>
 <h3>공지사항</h3>
 … 생략 …
 <h2>하단영역</h2>
 <h3>하단메뉴</h3>
 <ul>
  <li>회사소개</li>
  <li>개인정보보호방침</li>
  <li>사이트맵</li>
  … 생략 …
 </ul>
</body>
</html>

여기서 잠깐

h1은 웹 페이지에서 한 번만 사용해야 하나요? 또 어떤 콘텐츠를 h1으로 지정해야 하나요?

h1이 한 웹 페이지에 몇 번 나오는 것이 좋은가에 대해서는‘웹 페이지를 대표하는 역할로 보면 한 번만 사용하는 것이 좋다.’라는 의견과‘남용하지 않는 조건 아래 필요하면 두 번 이상 사용할 수 있다.’라는 의견으로 나누어집니다. 이는 마크업하면서 많은 논의가 이루어졌던 부분 중 하나이며, HTML 권고안에서는 횟수에 대해 명시되어 있지 않습니다.

이때 텍스트와 이미지를 제목 요소의 콘텐츠로 사용한 경우에는 읽어 주는 순서가 다릅니다. 외국에서 사용하는 스크린 리더 중 하나인 JAWS의 경우도 이와 같은데, 작업할 때 같은 레벨의 h 요소를 텍스트, 이미지 형식에 맞춰 주면 시각장애인이 스크린 리더로 웹을 탐색할 때 도움이 됩니다.

그리고 어떤 정보를 h1에 담아야 하는가에 대해서는‘웹 페이지를 대표하는 정보인 웹 사이트의 로고에 지정하는 것이 맞다.’라는 의견과‘웹 페이지의 메인 콘텐츠 제목에 사용해야 한다.’는 의견이 많았습니다. 이 이야기는 트위터(Twitter)에서도 제기된 후 h1debate라는 웹 사이트에서 투표도 진행되었습니다.

이러한 이야기가 진행되는 상황 속에서 2009년 가을 W3C의 w3.org 사이트가 리뉴얼되었습니다. 리뉴얼된 W3C 사이트의 h1은 어떻게 사용되었을까요? 서브 페이지에서 사용한 h1은 2개이며, 로고와 메인 타이틀의 두 곳에 각각 h1을 사용했습니다.

여기에서 W3C 사이트는 h1을 잘 사용하기 위한 하나의 예로 보고 페이지에 담긴 콘텐츠를 나타낼 수 있는 가장 중요한 항목에 사용했는지, 단계가 순차적인지, 너무 남용하지는 않았는지를 고려하여 사용하는 것이 중요합니다.
